SAVE SYSTEM
Our very simple take on a save system. This allowed us to run tests and to debug things very easily, making us able to ad a lot more content than we would've otherwise.
SAVE MANAGER: Saving

All of the Game Data is stored as pairs of 2 strings, a key and a value, inside of a Dictionary.

The Game Data is saved every time the player reaches a checkpoint.

If the player isn't on the Title screen, a save icon appears every time the save function is ran.

To save the game, we take everything inside of the Game Data Dictionary, we turn it into a JSON file, and we write it into a file at Unity's integrated 'persistent data' file location.

SAVE MANAGER: Loading

This function is ran as soon as the game is loaded, as a way to retrieve the player's Game Settings.

We start by trying to retrieve the Save Data JSON file.

If the Save File doesn't already exist, we create an empty one and fill it with Default Game Settings.

After retrieveing the Game Data from the file, we convert it from a JSON back into a Dictionary.

Lastly, we apply the retrieved Game Settings to make sure that everything is ready to go.

SAVE MANAGER: Persistent Data

At any point, any script can Write or Retrieve Persistent Data easily.

This data can be used at any point, by any object, and for any purpose.

In the context of this game, it is currently used for:

  • Game Settings
  • Progression Flags
  • Persistent Conditions for events
  • Remembering the last Checkpoint reached
  • Remembering the last Objective
